# Human Status Card Design

## Goal

把“当前草稿”状态卡从系统内部诊断面板，改成用户能一眼看懂的结论卡。

## Problem

- 当前卡片暴露了 `规则单页 / 实测预览 / 已隐藏低优先级内容` 这类系统术语。
- “通过”会误导用户，以为这份简历已经足够体面；但内容偏少时，虽然 technically 可以导出，实际观感仍会偏空。
- 用户真正关心的是：能不能导出、会不会空/满、下一步最值得补什么。

## Chosen Direction

- 卡片标题改成 `这版简历怎么样`。
- 右上角只保留一个轻状态：
  - `可直接导出`
  - `建议补强`
  - `建议压缩`
  - `先处理超页`
- 卡片正文固定成三层：
  - 结论：一句人话判断
  - 原因：为什么是这个判断
  - 下一步：最值得补或压的一步

## Decision Rules

- 内容偏少但单页稳定：
  - 允许导出
  - 文案改成 `可以导出，但建议先补强`
- 内容均衡且单页稳定：
  - 文案改成 `可以直接导出`
- 内容偏满但未超页：
  - 文案改成 `可以导出，但略显拥挤`
- 超页或导出被阻断：
  - 文案改成 `先压缩一下，再导出更稳`

## Notes

- 仍可保留系统消息和阻断原因，但只能作为辅助说明，不再成为卡片主文案。
- “通过 / 待检测 / 已隐藏 0 项” 等系统判断不再直接出现在用户主视图里。
