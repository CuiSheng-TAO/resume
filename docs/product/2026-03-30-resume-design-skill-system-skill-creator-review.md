# `skill-creator` 审查附录

**日期：**2026-03-30
**对象：**简历设计技能系统
**目的：**明确 `skill-creator` 在这套系统里的审查边界，避免把审美判断误当成结构判断。

## 1. 结论先行

这套简历设计技能系统可以接受 `skill-creator` 审查，但审查范围必须限定在“结构是否适合做成技能”这个层面。

`skill-creator` 适合看的是：

- 边界是否清楚
- 触发条件是否明确
- 复用性是否足够
- 结构是否适合后续独立调用

`skill-creator` 不负责看的是：

- 模板到底漂不漂亮
- 某个配色是否更高级
- 某个家族是否更有气质
- 某套模板的审美是不是“真的好看”

这两类判断不能混在一起。

## 2. 审查分工

### 2.1 `skill-creator` 适合审查什么

`skill-creator` 更像是一个“技能结构审查器”，它能很好地检查：

- 一个技能是不是讲清楚了自己负责什么
- 输入输出边界有没有被限定住
- 规则是不是足够稳定，能不能复用
- 有没有把太多设计判断塞进同一个技能里
- 哪些内容适合进入 skill，哪些更适合留在设计文档或参考资产里

对于这套简历设计系统来说，这非常有价值，因为我们需要先把“设计能力如何被组织”说清楚，再讨论哪些部分值得升级成真正可调用的 skill。

### 2.2 `skill-creator` 不适合审查什么

它不应该承担视觉评审的职责。

原因很简单：模板是否美、是否高级、是否有设计感，这些是审美问题，不是 skill 结构问题。`skill-creator` 可以帮我们判断一个能力能不能被稳定描述出来，但不能替我们判断这套视觉到底好不好看。

如果把这两件事混为一谈，就会出现一个问题：结构上看起来很完整，但审美上其实没有通过。

## 3. 建议保留为内部设计系统的 skill

建议当前这套内容**先保留为内部设计系统的 skill**，而不是立刻作为面向外部的可调用 skill 发布。

当前明确建议继续保留在内部设计系统里的 skill 包括：

- `版面节奏`
- `标题系统`
- `教育呈现`
- `经历可读性`
- `单页平衡`
- `冷静学术`
- `现代清爽`
- `重点鲜明`

这些名字都来自当前已经确认的 `12 个设计 skill`，不是这份附录临时新造出来的术语。它们之所以暂时保留在内部，是因为它们仍然更依赖项目语境、模板资产和审美判断，不适合马上抽成独立调用的 skill。

理由如下：

1. **它当前更像设计规范集合。** 这套内容的首要价值，是帮团队统一“什么叫好看、什么叫安全、什么叫可投递”，而不是立即对外暴露一个通用 API。
2. **它还包含较强的项目语境。** 例如校园招聘、模板家族、打印安全、家族差异，这些都和当前产品阶段强相关，适合先作为内部标准沉淀。
3. **它更适合先验证稳定性。** 先作为内部系统使用，可以继续观察哪些规则最常被复用、哪些规则最容易被误解、哪些规则最适合抽成更小的 skill。

## 4. 建议后续升级为真实可调用 skill 的候选

建议把这套系统里的少数最稳定部分，作为后续升级为真实可调用 skill 的候选。

优先候选包括：

- `信息层级`
- `信息压缩`
- `打印安全`
- `温暖专业`

其中前 3 项更适合升级，原因是它们更像规则，而不是某一时点的视觉偏好。`温暖专业` 是这里的一个特例：它虽然属于风格 skill，但已经有明确的审美参考底座和稳定的家族边界，后续更有机会被整理成“受控风格 skill”，而不是完全自由的视觉偏好。

它们的共同特点是：

- 触发语义清楚
- 适用范围相对稳定
- 复用价值高
- 不太依赖单个模板的具体外观

## 5. 为什么这样分层更合理

这样分层的好处是：

- 设计系统先把审美底座定住
- `skill-creator` 再判断哪些能力值得抽象成可复用技能
- 真正的美学判断留给人和设计规范
- 真正的结构判断交给 `skill-creator`

这会让系统更健康：

- 设计不会被工具误导
- 工具不会被迫承担自己不擅长的任务
- 后续升级时也更容易判断哪些部分已经成熟

## 6. 最终建议

建议把这份简历设计系统视为一个**内部设计系统 skill**，并把 `skill-creator` 用作它的**结构审查工具**。

最终升级路径应当是：

1. 先把内部设计系统稳定下来
2. 再用 `skill-creator` 识别哪些部分足够清晰、足够稳定、足够可复用
3. 最后只把最规则化的部分升级成真正可调用的 skill

这条路径的好处是，既保留了设计主导权，也保留了未来产品化、技能化的空间。
