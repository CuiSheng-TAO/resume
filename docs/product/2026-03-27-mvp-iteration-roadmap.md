# HR 陪跑式简历机 MVP 迭代路线

## 北极星

- 让中国大陆校招 / 应届生用户在 `3 分钟内` 得到一版 `可投递、单页、美观` 的中文简历草稿

## 迭代方法论

每一轮只解决一个最明显的瓶颈，并明确四件事：

1. `本轮目标`
   这一轮到底要解决哪一个关键问题。
2. `用户价值`
   用户能明显感知到什么变好。
3. `守门指标`
   哪些数据或验证结果证明这一轮可以进入下一轮。
4. `明确不做`
   哪些诱人的功能先不碰，避免 MVP 失焦。

## Iteration 1

- 名称：`AI 基础设施产品化`
- 目标：让真实大模型接入具备 `可配置、可回退、可限流、可观测` 的上线基础
- 用户价值：
  - AI 改写不再是脆弱 demo
  - 出错时仍然有本地 fallback，不会卡死
  - 产品能逐步切流、调成本、回滚 prompt
- 范围：
  - 环境变量与运行时配置收口
  - Prompt 版本化
  - Anthropic 调用超时与重试
  - JSON 提取容错
  - 轻量限流
  - 前端区分真实 AI / 本地回退状态
  - `.env.example`
- 不做：
  - 多模型调度
  - 成本面板
  - 真正的服务端日志平台接入
  - JD 定制
- 守门指标：
  - 无密钥时主流程仍可用
  - 超时或解析失败自动回退
  - 同 IP 高频请求会被限流
  - 测试、lint、build 全通过

## Iteration 2

- 名称：`单页引擎 V2`
- 目标：让“一页纸”从规则估算升级为真实版面平衡能力
- 用户价值：
  - 经历多时不粗暴缩小字号
  - 经历少时也不显得空与廉价
  - 隐藏内容与压缩策略更像专业 HR 的取舍
- 范围：
  - 内容评分层
  - DOM 真实高度测量
  - overflow / sparse 双策略
  - 用户锁定内容后的二次平衡
- 守门指标：
  - 高频样本的一页达成率明显提升
  - 内容少 / 内容多都能保持高级感

## Iteration 3

- 名称：`岗位针对性增强`
- 目标：让同一份求职资产可以按 JD 快速微调
- 用户价值：
  - 针对不同岗位导出不同版本
  - 不用从头改整份简历

## 发布节奏

- 每轮只发布一个主叙事
- 每轮结束都做一次：
  - 验收清单
  - 风险复盘
  - 下一轮瓶颈确认
